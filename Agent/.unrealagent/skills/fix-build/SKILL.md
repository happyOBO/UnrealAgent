이 작업은 **완료까지 연속으로 수행**한다. 에러를 찾기만 하고 멈추지 말고, 수정 적용 후
검증까지 끝낸다.

## 입력
사용자가 붙여넣은 컴파일러 출력(MSVC 에러 코드 C####). 에러 텍스트가 없으면 먼저
요청한다.

## 절차
1. **에러 파싱**: 각 에러의 파일·줄·에러코드·메시지를 추출한다. 첫 번째(근본) 에러부터
   처리한다 — 뒤따르는 에러는 연쇄일 때가 많다.
2. **원인 분류** (Shooter에서 자주 나온 유형):
   - `C1083 'X.h': No such file or directory` → include 경로/모듈 의존성 누락.
     올바른 헤더 경로 확인(예: `Kismet/KismetAnimationLibrary.h`)과 Build.cs의
     PublicDependencyModuleNames 점검.
   - `C2027 정의되지 않은 형식 'UX'` / `C2065` → 전방선언만 있고 정의 헤더 미포함.
     해당 타입의 실제 헤더를 include 한다(예: GameplayEffect.h).
   - `C2601 지역 함수 정의가 잘못됨` → 함수 시그니처/괄호 짝, `_Implementation` 누락 등.
   - `C2665 오버로드된 함수가 없음` (예: GetNameSafe) → 인자 타입 불일치. 올바른
     오버로드에 맞게 캐스팅/인자 수정.
3. **수정 적용**: 최소 변경으로 고친다. 기존 코드 스타일·include 순서를 따른다.
4. **검증**: 빌드 도구가 있으면 재빌드, 없으면 사용자에게 빌드를 요청하고 다음 에러를
   이어서 처리할 준비를 한다.
5. **보고**: 무엇이 원인이었고 어떻게 고쳤는지 에러별로 한 줄 요약.

## 주의
- 에러를 가리려고 임시 주석 처리/기능 제거로 때우지 말 것 — 근본 원인을 고친다.
- UE5.7 API 기준으로 판단한다(deprecated API 도입 금지).
